* Overview

Phase-by-phase plan for the Agda string-diagram tool under
~src/Categories/APROP/Hypergraph/~.  Each phase builds on the
previous one.  The ultimate goal is to make every module typecheck
with ~--safe~ and have ~CategoricalCrypto.agda~ transitively import
them.

* Active postulates

All three blocks live in files that currently omit ~--safe~ because
of these postulates.  The top-level ~CategoricalCrypto.agda~ does
*not* import any of them, so the library stays ~--safe~ in the
meantime.

** DONE [#A] ~ψ-elab-T~ — edge-label compatibility in ~hTensor-resp-≅ᴴ~
   :PROPERTIES:
   :file:     src/Categories/APROP/Hypergraph/Congruence.agda
   :difficulty: medium
   :END:

   Discharged.  The proof splits along ~splitAt G₁.nE e~ and for each
   branch chains six ~subst₂-trans~ collapses to thread ~α~ (unlock
   ~T₂.elab~ via ~hT₂.elab-c-inj₁~), ~β̄~ (cancel with
   ~subst₂-sym-subst₂~), ~γ~ (cross from G₂.elab to G₁.elab via
   ~IG.ψ-elab~), and ~δ~ (repackage to T₁.elab) equalities.

   Key trick for the ~--without-K~ constraint: the ~x~ argument of
   ~subst₂-trans~ is written as ~T₂.elab (inject+ K₂.nE (IG.ψ eG))~
   rather than ~T₂.elab (ψ-T e)~.  Inside the ~with~-aux body
   ~ψ-T e~ reduces to that form anyway (because the scrutinee
   substitution ~splitAt G₁.nE e ↦ inj₁ eG~ propagates through
   ~ψ-T~'s definition), but writing it in the reduced form avoids
   an inference failure where Agda spuriously reported
   ~G₂.nV != G₁.nV~ while trying to unify the implicit indices.

** DONE [#B] ~hCompose-resp-≅ᴴ~ — full theorem
   :PROPERTIES:
   :file:     src/Categories/APROP/Hypergraph/Congruence.agda
   :difficulty: hard
   :END:

   Discharged.  ~Congruence.agda~ is now ~--safe~.

   Executed plan:

   1. Refactored ~FromAPROP~ to expose a top-level
      ~hCompose-impl~ module (parallel to ~hTensor-impl~) holding
      ~remap'~, ~remap~, ~vlab-c~, ~ein-c~, ~eout-c~, ~elab-c~, and
      their reduction lemmas.  ~hCompose~ is now a thin wrapper that
      reexports via ~open hCompose-impl G K~.

   2. Proved ~remap'-comm~ by induction on ~(K.dom, G.cod)~ with a
      four-way ~with v Fin.≟ k | IK.φ v Fin.≟ IK.φ k~ split.  The
      impossible yes/no and no/yes cases are discharged via
      injectivity of ~IK.φ~, which is itself derivable from
      ~IK.φ-left~ by the one-line
      ~trans (sym (φ-left v₁)) (trans (cong φ⁻¹ eq) (φ-left v₂))~.

   3. Plumbed ~remap-comm~ through ~cod-C~, the K-branches of
      ~atom-ein-C~, ~atom-eout-C~, and ~ψ-elab-C~ via
      ~map-via-remap~ (the hCompose analog of ~map-via-raise~).

   4. Finished ~ψ-elab-C~ with the same six-step ~subst₂-trans~
      chain used for ~ψ-elab-T~, using the reduced forms
      ~C.elab (inject+ K.nE (IG.ψ eG))~ and
      ~C.elab (raise G.nE (IK.ψ eK))~ for the ~x~ parameter.

** BLOCKED [#C] ~soundness-axiom~ — 18 atomic ≈Term axioms
   :PROPERTIES:
   :file:     src/Categories/APROP/Hypergraph/Soundness.agda
   :line:     65
   :difficulty: blocked
   :END:

   The catch-all in ~Soundness.agda~ absorbs 18 constructors.  By
   vertex-count behaviour after translation:

   *Group (a):* LHS and RHS have equal vertex count (only ~+~ re-
   association or shape-swap differs).  Provable with the current
   ~_≅ᴴ_~ but the bijection proofs are still non-trivial (~200+
   lines each, similar to ~hCompose-resp-≅ᴴ~).

   - ~assoc~ — ~(a+b)+c = a+(b+c)~ both sides.
   - ~⊗-∘-dist~ — ~(a+b)+(a'+b')~ vs ~(a+a')+(b+b')~.
   - ~hexagon~ — three 3-fold compositions, equal total.

   *Groups (b), (c):* the other 15 axioms.  LHS has strictly more
   vertices than RHS (or the two sides have different counts
   depending on the APROP signature), because ~hCompose~ keeps
   the K-side vertex set after identifying ~K.dom~ with ~G.cod~
   via ~remap~.

   Example: ~idˡ : id ∘ f ≈Term f~.  At ~f : HomTerm A B~,
   - LHS ~⟪id ∘ f⟫ = hCompose ⟪f⟫ (hId B)~ has ~⟪f⟫.nV + length(flatten B)~ vertices.
   - RHS ~⟪f⟫~ has ~⟪f⟫.nV~ vertices.

   Since ~length(flatten B) > 0~ whenever ~B~ has any atom, the
   two cannot be in bijection; no refinement of the bijection
   functions can fix this.

   Discharging these axioms therefore requires one of the
   following architectural changes before the lemmas themselves
   can be written.

*** TODO Option A — prune dangling vertices in ~hCompose~
    Change ~hCompose~ to delete any K-side vertex that is in
    ~K.dom~ (since such a vertex has been identified with a G-side
    vertex via ~remap~).  After pruning, ~idˡ~ / ~idʳ~ produce
    exactly ~⟪f⟫.nV~ vertices.  Downside: ~remap~ must be rewritten
    to produce a vertex in the pruned space, and ~elab-c~ needs to
    dispatch through the pruning map.

    *Implementation sketch:*

    1. Prove that every ~⟪f⟫~ (f : HomTerm A B) has a ~dom~ and
       ~cod~ with pairwise-distinct entries.  By induction on the
       primitive constructors (hEmpty, hVar, hGen, hSwap, hId) and
       preservation under hTensor/hCompose.  Record as a lemma
       ~⟪_⟫-distinct-dom/cod~ in FromAPROP (or in a new
       ~Hypergraph.Invariant~ module).

    2. Add a new ~hCompose-impl~ variant (parameterized by the
       distinctness proof of ~K.dom~) with:

       - ~nV' = G.nV + (K.nV - length K.dom)~.

       - ~compress : Fin K.nV → Fin (K.nV - length K.dom + length K.dom)~
         moving K-dom vertices to the "tail" ~length K.dom~
         positions, non-K-dom to the "head".

       - ~remap' : Fin K.nV → Fin (G.nV + (K.nV - length K.dom))~
         that sends each K.dom vertex to ~injL G.cod[i]~ and each
         non-K.dom vertex ~v~ to ~injR (compress v)~.

    3. Rewrite the ~hCompose~-branch of ~elab-c~, ~ψ-elab-C~, and
       ~boundary-eq~ to use the new ~remap'~.

    4. Port ~Congruence.hCompose-resp-≅ᴴ~ to the new ~hCompose~.
       The structure is the same (six-step ~subst₂-trans~ chain)
       but the indexing in Fin changes.

    5. Discharge the 15 "LHS > RHS" axioms using vertex bijections
       with matching counts (all but the 3 group-(a) axioms).

*** TODO Option B — coarsen ~_≅ᴴ_~ to allow vertex merging
    Replace the strict vertex bijection with a pair
    ~(h : Fin G.nV → Fin K.nV, coh : list-of-merges)~ expressing
    that ~G~ is obtained from ~K~ by identifying some vertices.
    Preserves the existing ~hCompose~ but makes the iso relation
    heavier.  Downside: ~refl-≅ᴴ~ / ~sym-≅ᴴ~ / ~trans-≅ᴴ~ all need
    to be rewritten, and the structural-congruence proofs in
    ~Congruence.agda~ will need to be reworked.

*** Recommendation
    Option A is the lesser refactor and matches the TensorRocq
    paper (Section 3.3 "canonical hypergraph").  Do A once the
    simpler ~ψ-elab-T~ / ~hCompose-resp-≅ᴴ~ postulates are
    discharged, since A changes ~hCompose~ and any proofs written
    against the old ~hCompose~ will need porting.

    The three group-(a) axioms (~assoc~, ~⊗-∘-dist~, ~hexagon~)
    could in principle be discharged without Option A, but their
    proofs will need to be rewritten anyway if/when ~hCompose~
    changes.  So start with Option A.
