* Overview

Phase-by-phase plan for the Agda string-diagram tool under
~src/Categories/APROP/Hypergraph/~.  Each phase builds on the
previous one.  The ultimate goal is to make every module typecheck
with ~--safe~ and have ~CategoricalCrypto.agda~ transitively import
them.

See https://arxiv.org/pdf/2604.17592 for high-level details.

* Active postulates

All three blocks live in files that currently omit ~--safe~ because
of these postulates.  The top-level ~CategoricalCrypto.agda~ does
*not* import any of them, so the library stays ~--safe~ in the
meantime.

     ПғвҲҳ[fвҠ—g]вүҲ[gвҠ—f]вҲҳПғ (Пғ-nat), hexagon, pentagon, assoc, вҠ—-вҲҳ-dist.
     See "Refactor / 6." below for the per-axiom proof plan.

   Dispatch uses ~{-# INJECTIVE_FOR_INFERENCE вҹӘ_вҹ« #-}~ (in
   ~Translation.agda~) plus ~--lossy-unification~ (on
   ~Soundness.agda~).  Without both flags, dispatching any focused
   postulate whose type mentions a `hComposeP ... (hTensor ...)`
   chain triggers a ~25-minute unification blow-up.  With both,
   every dispatch typechecks in <1s.

   Historical note (pre-~hComposeP~): the catch-all used to absorb
   18 constructors.  With the switch to pruned composition
   (~hComposeP~), the "LHS has more vertices than RHS" problem
   disappeared, and all axioms became reachable at the вү…бҙҙ level.

   Example: ~idЛЎ : id вҲҳ f вүҲTerm f~.  At ~f : HomTerm A B~,
   - LHS ~вҹӘid вҲҳ fвҹ« = hCompose вҹӘfвҹ« (hId B)~ has ~вҹӘfвҹ«.nV + length(flatten B)~ vertices.
   - RHS ~вҹӘfвҹ«~ has ~вҹӘfвҹ«.nV~ vertices.

   Since ~length(flatten B) > 0~ whenever ~B~ has any atom, the
   two cannot be in bijection; no refinement of the bijection
   functions can fix this.

   Discharging these axioms therefore requires one of the
   following architectural changes before the lemmas themselves
   can be written.

* Refactor

  Strategic refactoring plan to discharge the remaining 12 вүҲTerm
  axioms efficiently by maximizing reuse.  The key observation is
  that both `idЛЎ-proof` and `ПғвҲҳПғ-proof` contain reasoning that only
  uses record fields of their `G`/`K` arguments, not any APROP-term
  structure.  Generalizing these to take arbitrary hypergraphs
  (rather than ~вҹӘfвҹ«~-specific ones) creates reusable building
  blocks for most remaining axioms.

  The sequence below is ordered by effort-to-unlock ratio: each
  item unlocks downstream axioms and is more tractable given the
  preceding items.

** DONE [#A] 1. Refactor ~idЛЎ-proof~ вҶ’ ~hCompose-hId-R-iso~
