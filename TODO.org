* Overview

Phase-by-phase plan for the Agda string-diagram tool under
~src/Categories/APROP/Hypergraph/~.  Each phase builds on the
previous one.  The ultimate goal is to make every module typecheck
with ~--safe~ and have ~CategoricalCrypto.agda~ transitively import
them.

* Active postulates

All three blocks live in files that currently omit ~--safe~ because
of these postulates.  The top-level ~CategoricalCrypto.agda~ does
*not* import any of them, so the library stays ~--safe~ in the
meantime.

** BLOCKED [#C] ~soundness-axiom~ — 18 atomic ≈Term axioms
   :PROPERTIES:
   :file:     src/Categories/APROP/Hypergraph/Soundness.agda
   :line:     65
   :difficulty: blocked
   :END:

   The catch-all in ~Soundness.agda~ absorbs 18 constructors.  By
   vertex-count behaviour after translation:

   *Group (a):* LHS and RHS have equal vertex count (only ~+~ re-
   association or shape-swap differs).  Provable with the current
   ~_≅ᴴ_~ but the bijection proofs are still non-trivial (~200+
   lines each, similar to ~hCompose-resp-≅ᴴ~).

   - ~assoc~ — ~(a+b)+c = a+(b+c)~ both sides.
   - ~⊗-∘-dist~ — ~(a+b)+(a'+b')~ vs ~(a+a')+(b+b')~.
   - ~hexagon~ — three 3-fold compositions, equal total.

   *Groups (b), (c):* the other 15 axioms.  LHS has strictly more
   vertices than RHS (or the two sides have different counts
   depending on the APROP signature), because ~hCompose~ keeps
   the K-side vertex set after identifying ~K.dom~ with ~G.cod~
   via ~remap~.

   Example: ~idˡ : id ∘ f ≈Term f~.  At ~f : HomTerm A B~,
   - LHS ~⟪id ∘ f⟫ = hCompose ⟪f⟫ (hId B)~ has ~⟪f⟫.nV + length(flatten B)~ vertices.
   - RHS ~⟪f⟫~ has ~⟪f⟫.nV~ vertices.

   Since ~length(flatten B) > 0~ whenever ~B~ has any atom, the
   two cannot be in bijection; no refinement of the bijection
   functions can fix this.

   Discharging these axioms therefore requires one of the
   following architectural changes before the lemmas themselves
   can be written.

* Refactor

  Strategic refactoring plan to discharge the remaining 12 ≈Term
  axioms efficiently by maximizing reuse.  The key observation is
  that both `idˡ-proof` and `σ∘σ-proof` contain reasoning that only
  uses record fields of their `G`/`K` arguments, not any APROP-term
  structure.  Generalizing these to take arbitrary hypergraphs
  (rather than ~⟪f⟫~-specific ones) creates reusable building
  blocks for most remaining axioms.

  The sequence below is ordered by effort-to-unlock ratio: each
  item unlocks downstream axioms and is more tractable given the
  preceding items.

** TODO [#A] 1. Refactor ~idˡ-proof~ → ~hCompose-hId-R-iso~
   :PROPERTIES:
   :effort: 30 min
   :unlocks: generalized left-identity-composition
   :END:

   Current `idˡ-proof` module is parameterized by
   `(f : HomTerm A B)` but only uses `G = ⟪f⟫`'s record fields
   (nV, vlab, dom, cod, ein, eout, elab, …).  It does NOT depend
   on `f`'s syntactic structure.  Refactor the module header to:
   #+begin_src agda
     module hCompose-hId-R-proof
       {As : List X} {B : ObjTerm}
       (G : Hypergraph FlatGen As (flatten B))
       where
         ...
   #+end_src
   yielding a generic
   ~hCompose-hId-R-iso : hComposeP G (hId B) ≅ᴴ G~.

   `idˡ-sound` becomes a one-line specialization:
   `idˡ-sound f = hCompose-hId-R-iso ⟪f⟫`.

   The 230-line body of ~idˡ-proof~ transfers verbatim because
   all its helpers (cn≡0 via `hId-count-non-dom`, nE≡0 via
   `hId-nE`, `idˡ-cod-helper` via `hId-cod≡dom` +
   `hId-dom-Unique` + cast arithmetic, …) depend on `B`, not `f`.

   *Why this matters*: downstream axioms (λ-nat, triangle, some
   coherences) compose ⟪f⟫'s that are themselves complex
   expressions (e.g. `hTensor hEmpty ⟪f⟫` for `id⊗f`).  Having a
   generic `hCompose-hId-R-iso` lets us apply the identity law to
   those derived hypergraphs without re-deriving the proof.

** TODO [#A] 2. Prove ~hTensor-hEmpty-G-iso~
   :PROPERTIES:
   :effort: 2 h
   :unlocks: λ-naturality via chaining; partial progress on ρ-nat, triangle
   :END:

   Generalize `hTensor-hEmpty-hId-iso` from `hId A` to an
   arbitrary G:
   #+begin_src agda
     hTensor-hEmpty-G-iso
       : ∀ {As Bs} (G : Hypergraph FlatGen As Bs)
       → hTensor hEmpty G ≅ᴴ G
   #+end_src

   (definitional).

   - φ = φ⁻¹ = id.  Bijection laws = refl.
   - ψ = ψ⁻¹ = id.  Bijection laws = refl.
   - φ-lab = refl (splitAt 0 always gives inj₂, matches G.vlab).
   - ψ-ein / ψ-eout: routine `map-∘ / map-cong / map-id` (same
     as `hTensor-hEmpty-hId-iso`'s existing chain).
   - φ-dom / φ-cod: same routine chain.

   *Hard field: ψ-elab*.  Unlike the `hId A` specialization
   (where ψ-elab is vacuous because hId has no edges), a generic

   unfolds to `subst₂ FlatGen (map-via-raise vlab-injR …) … (G.elab e)`.
   Use the "strategic atom-ein" trick from `idˡ-proof`: choose
   `atom-ein = map-via-raise vlab-injR (G.ein e)` so the ψ-elab
   equation reduces to `refl` via the `with splitAt G.nE e` match.

   *Also provide mirror lemma* `hTensor-G-hEmpty-iso-substed`
   (see item 5).

** TODO [#A] 3. Prove ~hCompose-hId-L-iso~
   :PROPERTIES:
   :effort: 4 h
   :unlocks: idʳ directly; half of λ-nat, ρ-nat, α-nat via chaining
   :END:

   Port the `σ∘σ-proof` classify + mirror-witness structure to an
   arbitrary right-side hypergraph K:
   #+begin_src agda
     hCompose-hId-L-iso
       : ∀ {A : ObjTerm} {Bs} {K : Hypergraph FlatGen (flatten A) Bs}
       → Unique (Hypergraph.dom K)          -- side condition
       → hComposeP (hId A) K ≅ᴴ K
   #+end_src

   The side condition is supplied by `⟪_⟫-dom-unique` at the
   ~idʳ~ call site.

   *Structure* (template = `σ∘σ-proof` with G := hId A,
   K := arbitrary-with-Unique-dom):

   - C.nV = `(hId A).nV + count-non K.dom`.  φ splits via
     `splitAt (hId A).nV`:
     * inj₁ i ↦ `lookup K.dom (cast len-eq i)` (mirror into K).
     * inj₂ j ↦ `lookup (nonMem K.dom) j` (pruned-K-side).
     where `len-eq : (hId A).nV ≡ length K.dom` (via
     `hId-nV≡len-flatten` + `K.dom-ok` + `length-map`).

   - φ⁻¹ via `classify K.dom`:
     * inj₁ i ↦ `inject+ _ (cast (sym len-eq) i)`.
     * inj₂ j ↦ `raise (hId A).nV j`.

   - `φ-left`: case on splitAt; `classify-lookup-Unique` and
     `classify-lookup-nonMem` (newly added!) close the two
     branches.

   - `φ-rght`: case on classify; `classify-inj₁-lookup` and
     `classify-inj₂-lookup` close the two branches.

   - `φ-lab`: for the `inj₁ i` branch, the label chain is:
       `G.vlab i = (hId A).vlab i`
         `≡ lookup (flatten A) (cast _ i)`      [hId-vlab-lookup]
         `≡ K.vlab (K.dom[cast _ i])`           [K.dom-ok, pointwise]
       which matches K.vlab ∘ (lookup K.dom ∘ cast).
     For the `inj₂ j` branch, both sides equal
     `K.vlab (lookup (nonMem K.dom) j)`.

   - `φ-dom`: `C.dom = map injL (hId A).dom = map injL (range _)`
